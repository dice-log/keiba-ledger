# 時系列オッズ（0B41/0B42）取得・投入ガイド

JV-Data 0B41（単複枠）, 0B42（馬連）の時系列オッズを取得し、DB に格納する手順。

## 概要

| 項目 | 内容 |
|------|------|
| データ種別 | 0B41: 単複枠（単勝・複勝・枠連）, 0B42: 馬連 |
| 提供期間 | 1年間 |
| 更新タイミング | 発売以降、5〜7分ごと程度 |
| 前提 | 32bit Python + JV-Link（JRAvan） |

## パイプライン

```
1. fetch_timeseries_all.py   … JV-Link から取得 → JSONL
2. load_timeseries_to_db.py  … JSONL → raw.odds_timeseries_raw
3. normalize_odds_timeseries.py … raw → analytics.odds_timeseries
```

## 1. 過去分一括取得

```powershell
# 2023年〜2024年分を取得（年度ごとに実行）
py -3.11-32 scripts/fetch/fetch_timeseries_all.py --from 2023-01-01 --to 2024-12-31

# 2024年分のみ
py -3.11-32 scripts/fetch/fetch_timeseries_all.py --from 2024-01-01

# O1（単複枠）のみ
py -3.11-32 scripts/fetch/fetch_timeseries_all.py --from 2024-01-01 --only-o1

# テスト（100件のみ）
py -3.11-32 scripts/fetch/fetch_timeseries_all.py --from 2024-01-01 --limit 100
```

出力: `data/timeseries_YYYYMMDD.jsonl`（年度ごとに1ファイル）

## 2. raw テーブルへ投入

```powershell
# 最新の timeseries_*.jsonl を使用
python scripts/fetch/load_timeseries_to_db.py

# 指定ファイル
python scripts/fetch/load_timeseries_to_db.py data/timeseries_20240101.jsonl
```

## 3. analytics テーブルへ正規化

```powershell
python scripts/transform/normalize_odds_timeseries.py

# ドライラン（DB書き込みなし）
python scripts/transform/normalize_odds_timeseries.py --dry-run
```

## テーブル構成

### raw.odds_timeseries_raw

生データ。`dataspec`, `record_type`, `raw_text` を保存。

### analytics.odds_timeseries

正規化済み。レース×時刻×組合せのオッズ。

| カラム | 説明 |
|--------|------|
| race_id | レースID |
| bet_type | win / place / bracket / quinella |
| combination | 組合せ（馬番・枠番など） |
| odds | オッズ |
| recorded_at | 記録時刻 |
| minutes_to_start | 発走まで何分前（未計算時は NULL） |

## 特徴量での利用

時系列データ投入後、ML で以下の特徴量が利用可能:

| 特徴量 | 説明 |
|--------|------|
| odds_ts_change_rate | (直前 - 公開) / 公開。オッズの急騰・急落 |

学習時に `--use-timeseries` を指定:

```powershell
python scripts/ml/train.py --use-timeseries
```

追加パターン（今後試す例）:
- 直前オッズそのもの (odds_ts_last)
- 公開時オッズ (odds_ts_first)
- 変動の大きさ（分散など）

`scripts/ml/features.py` の `TIMESERIES_FEATURE_COLS` に列を追加してパターンを試す。
